Route Optimizer

Our project aims to find the most optimal route between multiple locations. The main difference between our program and other navigation programs (ex. Google Map) is that our program not only finds the shortest route from point A to point B, but it also optimizes the order in which multiple locations should be visited for minimal road length. Our aim is that our project will be used by businesses which need to visit a large number of different locations as it will cut down their logistic expenses and will save them time. It can also be used by individuals who need to visit many locations at one time, like when they want to tour many places on holidays. The program uses approximation algorithms used in optimizing the Traveling Salesmen Problem to calculate the optimized route. To use this program, the user should first give the locations by either selecting from the map or directly giving the coordinates. Then, the user can apply some conditions to the algorithm such as selecting the location to go to last or for example forcing the algorithm to go to the market and before coming home (assuming home and the market are entered locations) finally, the optimized route between the locations are computed and shown to the user in a map.
